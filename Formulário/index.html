  1. Na tag < input> com atributo type="text", para que serve o atributo "placeholder"?

    2. O atributo placeholder é uma pequena frase ou palavra, que tem o intuito de ajudar usuários a entender como devem preencher determinado formulário, inserindo um texto contextual no campo a ser respondido

  2. Na tag < input> com atributo type="radio" ou type="radio", para que serve o atributo "checked"?

    3. O checked atributo é um atributo booleano. Quando presente, ele especifica que um < input> elemento deve ser pré-selecionada (marcada), quando a página é carregada. O checked atributo pode ser usado com < input type="checkbox"> e < input type="radio"> .

  3. Na tag < input> para que serve o atributo "value"?

    4. O que faz o atributo value no HTML? O value é a informação que será enviado pro servidor, pode ser usado 1, 2, f, m, masculino, feminino etc. Então quem vai receber essa informação é quem definiu o value.

  4. Que atributo deve ser utilizado para agrupar vários botões de rádio em um formulário de modo que apenas um deles fique marcado de cada vez?

    5. A tag < button> O HTML Button pode ser definido como um atributo pela tag de abertura < button> e a tag de fechamento < /button>. Dentro de um elemento < button>, você pode colocar conteúdo, como texto ou imagens.

  5. Para que serve a tag < label>? Para que serve o atributo "for" nessa tag?

    6. A tag label é útil no trabalho com formulários. Ela especifica qual o "rótulo" do input (a que se refere o input), e ajuda na experiência do usuário durante a utilização e preenchimento do formulário. O atributo for faz um tipo de link entre a label e o input . Logo o for tem que apontar para o ID do input .

  6. Na tag < option> para que serve o atributo "selected"?

    7. Como usar a tag option no HTML? Em um formulário Web, o elemento HTML < option> é usado para criar um controle que representa um item dentro de um elemento HTML5 < select> , < optgroup> ou < datalist> Se presente, este atributo booleano indica que a opção está selecionada inicialmente. Se o elemento < option> for descendente de um elemento < select> cujo atributo multiple não está definido, apenas um único < option> deste elemento < select> pode ter o atributo selected.

  7. Nas tags de formulário para que serve o atributo "required"?

    8. Esse é um atributo booleano usado para indicar que um determinando campo de formulário é obrigatório para o envio do mesmo. Ao adicionar esse atributo a um campo de formulário, o navegador obriga o usuário a inserir dados naquele campo antes de enviar o formulário.


  8. Na tag < form> para que serve o atributo "action"?

    9. O URI de um programa que processa as inofrmações por meio do formulário. Esse valor pode ser substituído por um atributo "formaction" em um elemento < buttom> ou < imput>

  9. Na tag < form> para que serve o atributo "method"?

    10. O atributo method define o método HTTP com que o formulário HTML irá lidar com os dados recebidos. São eles: o método GET e o método POST.

  10. Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, qual deve ser o valor do atributo "method" na tag < form>?

    11. O método GET é usado pelo navegador para solicitar que o servidor envie de volta um determinado recurso. Portanto, é como se falássemos ao servidor “Servidor, eu quero obter este recurso.”. Nesse caso, o navegador envia um corpo vazio. Portanto, já que o corpo fica vazio, se um formulário for enviado através do método GET, os dados serão reconhecidos pelo servidor através da URL.

  11. Qual valor deve ser informado no atributo "method" na tag < form> em um formulário que será usado para fazer upload de arquivos?

    12. O método POST é utilizado no navegador para conversar com o servidor. Os dados são enviados ao servidor através do corpo da solicitação HTTP. Também é realizada uma solicitação de resposta. Portanto, é como se falássemos ao servidor “Servidor, verifique esses dados e me retorne um resultado adequado”. Dessa forma, ao enviar um formulário através do método POST, os dados serão anexados ao corpo da solicitação HTTP.Diferentemente do método GET, explicado anteriormente, o método POST não inclui o corpo na URL. Portanto, os dados enviados não ficarão visíveis na URL.
